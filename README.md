# PlayDreams Code Templates

Game code templates for the PlayDreams platform. These templates provide complete, working Godot games that serve as starting points when a user's game description matches the template's genre.

## Templates

| Template | Genre | Description |
|----------|-------|-------------|
| `bubble-shooter` | Casual/Puzzle | Physics-based bubble popping game with hexagonal grid |
| `match3-classic` | Casual/Puzzle | Classic match-3 puzzle game with cascading mechanics |

## Template Structure

Each template directory contains:
- `template.json` - Metadata for matching and configuration
- `project.godot` - Godot project file
- `scenes/` - Game scene files
- `scripts/` - GDScript game logic
- `assets/` - Sprites, sounds, and other assets
- `resources/` or `config/` - Game configuration resources

## Adding New Templates

1. Create a new directory with a kebab-case ID (e.g., `tower-defense`)
2. Copy your Godot game files (scenes, scripts, assets)
3. **CRITICAL**: Delete all `*.import` files (they contain machine-specific UIDs)
4. Remove: `.godot/`, `web-build/`, `.git/`, `*.apk`, temp files
5. Create `template.json` with metadata (see schema below)
6. Commit and push

## Template Metadata Schema

```json
{
  "id": "template-id",
  "name": "Template Display Name",
  "version": "1.0.0",
  "genreIds": ["genre/sub-genre", "genre"],
  "keywords": ["keyword1", "keyword2"],
  "dimension": "2d",
  "godotVersion": "4.5",
  "license": "MIT",
  "features": ["feature1", "feature2"],
  "coreFiles": ["scripts/main.gd", "scripts/core.gd"],
  "customizableAreas": [
    {
      "area": "visual-style",
      "files": ["assets/", "scripts/config.gd"],
      "description": "Sprites, colors, visual theme",
      "priority": "high"
    }
  ]
}
```

## Important Notes

- **No `.import` files**: These are regenerated by Godot on first open
- **No `.godot/` directory**: Contains local editor state
- **No `web-build/` directory**: Generated during export
- Templates must be Godot 4.5 compatible
